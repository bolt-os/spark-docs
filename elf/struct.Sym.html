pub const fn value(&self) -> u64

Returns the symbol’s value

This value is interpreted slightly differently depending on the type of file:

  • Relocatable files: An offset from the begging of the symbol’s associated [section]. If the symbol refers to the common section (see [is_common()]), this value instead holds alignment constraints for the symbol.

  • Executable and Shared Object files: To make symbols more useful for the dynamic linker, this value contains a virtual memory address instead of a section offset. The associated section is irrelevant in this case.
